Capítulo. Modelagem de banco de dados - conceitual.

A modelagem de banco de dados conceitual é a primeira etapa do processo de modelagem, onde o foco está na compreensão dos requisitos do sistema e na criação de uma representação abstrata dos dados, independente de tecnologia de banco de dados. O objetivo é capturar os conceitos e as regras de negócio do domínio em questão. Aqui estão os passos envolvidos na modelagem de banco de dados conceitual:

Item. 1. Coleta de requisitos: Nesta etapa, é fundamental compreender os requisitos do sistema, as necessidades dos usuários e as restrições do ambiente. Isso envolve entrevistas com stakeholders, análise de documentação e levantamento de informações relevantes.

Item. 2. Identificação de entidades: Com base nos requisitos coletados, identifique as entidades principais do sistema. As entidades são objetos do mundo real que serão representados no banco de dados. Por exemplo, em um sistema de vendas, as entidades podem incluir Cliente, Produto e Pedido.

Item. 3. Definição de atributos: Para cada entidade identificada, identifique os atributos que descrevem as características dos objetos. Os atributos são as propriedades ou informações que são relevantes para cada entidade. Por exemplo, para a entidade Cliente, os atributos podem ser nome, endereço e número de telefone.

Item. 4. Estabelecimento de relacionamentos: Identifique os relacionamentos entre as entidades. Os relacionamentos representam as associações ou interações entre as entidades. Eles podem ser do tipo um-para-um, um-para-muitos ou muitos-para-muitos. Por exemplo, um cliente pode fazer vários pedidos (um-para-muitos) e um pedido pode ter vários produtos (muitos-para-muitos).

Item. 5. Definição de restrições e regras de negócio: Durante a modelagem conceitual, é importante identificar as restrições e regras de negócio que se aplicam aos dados. Isso pode incluir restrições de integridade, restrições de domínio, restrições de cardinalidade, entre outras. Essas restrições garantem que os dados sejam consistentes e válidos.

Item. 6. Diagrama de Entidade-Relacionamento (DER): O resultado da modelagem conceitual é frequentemente representado por um diagrama de Entidade-Relacionamento (DER). O DER é uma representação visual das entidades, atributos e relacionamentos, facilitando a compreensão e a comunicação do modelo com os stakeholders.

A modelagem de banco de dados conceitual é uma etapa crucial para entender o domínio do sistema e capturar os requisitos de informação. Ela fornece uma visão de alto nível dos dados e serve como base para a modelagem lógica e física posterior. Um modelo conceitual bem elaborado ajuda a garantir a precisão, a integridade e a adequação dos dados em um sistema de banco de dados.
