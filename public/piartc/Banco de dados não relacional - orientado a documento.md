Capítulo. Banco de dados não relacional - orientado a documento.

Um banco de dados não relacional orientado a documento é um tipo de banco de dados NoSQL (Not Only SQL) que armazena e recupera dados em formato de documentos. Diferentemente dos bancos de dados relacionais, que organizam os dados em tabelas com estruturas predefinidas, os bancos de dados orientados a documento permitem que os dados sejam armazenados em documentos flexíveis, como JSON ou BSON (Binary JSON). Esses documentos podem ter estruturas diferentes, o que os torna especialmente adequados para dados sem uma estrutura fixa.

Aqui estão algumas características importantes dos bancos de dados não relacionais orientados a documento:

Item. 1. Modelo de dados flexível: Os bancos de dados orientados a documento permitem que os documentos sejam criados e modificados sem a necessidade de uma estrutura predefinida. Isso oferece flexibilidade para armazenar dados com diferentes formatos e estruturas, sem a necessidade de um esquema rígido.

Item. 2. Armazenamento de dados hierárquicos: Os documentos podem conter estruturas aninhadas e hierárquicas, permitindo representar relacionamentos complexos entre os dados.

Item. 3. Escalabilidade horizontal: Os bancos de dados orientados a documento são projetados para dimensionar horizontalmente, o que significa que podem lidar com grandes volumes de dados distribuindo-os em vários servidores.

Item. 4. Consultas flexíveis: Os bancos de dados orientados a documento geralmente oferecem recursos de consulta flexíveis, permitindo realizar consultas complexas e filtrar dados com base em critérios específicos. Alguns bancos de dados também suportam consultas baseadas em índices secundários para melhorar o desempenho das consultas.

Item. 5. Desempenho: Devido à sua estrutura de armazenamento otimizada para documentos, os bancos de dados orientados a documento podem oferecer bom desempenho para operações de leitura e gravação.

Item. 6. Alta disponibilidade: Muitos bancos de dados orientados a documento são projetados com recursos de replicação e distribuição para garantir alta disponibilidade dos dados.

Alguns exemplos populares de bancos de dados não relacionais orientados a documento incluem o MongoDB, o Couchbase e o Apache Cassandra. Esses bancos de dados são amplamente utilizados em cenários onde a flexibilidade do modelo de dados e a escalabilidade horizontal são essenciais, como aplicações web, gerenciamento de conteúdo, análise de dados e microsserviços.
