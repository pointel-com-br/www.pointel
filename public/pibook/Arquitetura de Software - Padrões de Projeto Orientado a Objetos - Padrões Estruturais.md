# Arquitetura de Software - Padrões de Projeto Orientado a Objetos - Padrões Estruturais

Na arquitetura de software, os padrões de projeto orientados a objetos desempenham um papel importante no projeto de sistemas eficientes e flexíveis. Os padrões estruturais são um grupo de padrões de projeto que se concentram na composição de classes e objetos para formar estruturas maiores e mais complexas. Eles fornecem soluções para organizar e gerenciar relacionamentos entre objetos.

Aqui estão alguns exemplos de padrões estruturais amplamente utilizados:

1. Adapter (Adaptador): O padrão Adapter permite que objetos com interfaces incompatíveis trabalhem juntos. Ele converte a interface de uma classe em outra interface que o cliente espera. Isso facilita a integração de componentes existentes e promove a reutilização de código.

2. Decorator (Decorador): O padrão Decorator adiciona responsabilidades adicionais a um objeto dinamicamente, envolvendo-o em um objeto decorador especial. Ele fornece uma alternativa flexível à criação de subclasses para estender funcionalidades, permitindo que os objetos sejam decorados com novos comportamentos em tempo de execução.

3. Composite (Composto): O padrão Composite permite tratar objetos individuais e coleções de objetos de maneira uniforme. Ele define uma estrutura de árvore hierárquica, onde os objetos individuais e os grupos de objetos são tratados de maneira recursiva. Isso permite que os clientes trabalhem com objetos individuais e com composições complexas de objetos de maneira transparente.

4. Bridge (Ponte): O padrão Bridge desacopla uma abstração da sua implementação, permitindo que ambas variem independentemente. Ele fornece uma estrutura na qual as hierarquias de abstração e implementação podem evoluir separadamente, facilitando a flexibilidade e a extensibilidade do sistema.

5. Facade (Fachada): O padrão Facade fornece uma interface unificada simplificada para um conjunto de interfaces em um subsistema. Ele oculta a complexidade subjacente do sistema, fornecendo uma interface mais simples e fácil de usar. Isso facilita o uso do subsistema e promove o desacoplamento entre os clientes e os componentes internos.

6. Flyweight (Peso-Leve): O padrão Flyweight visa minimizar o uso de memória compartilhando o máximo possível de estado entre objetos semelhantes. Em vez de criar novas instâncias para cada objeto, ele reutiliza objetos existentes, economizando recursos e melhorando o desempenho.

Esses são apenas alguns exemplos de padrões estruturais. Cada padrão aborda um problema específico relacionado à organização e à composição de classes e objetos. A aplicação adequada dos padrões estruturais pode promover a reutilização de código, o desacoplamento, a flexibilidade e a escalabilidade do sistema.
