# Quais as principais diretivas do Apache?

O Apache HTTP Server possui uma ampla variedade de diretivas que podem ser usadas para configurar e personalizar o comportamento do servidor. Aqui estão algumas das principais diretivas do Apache:

1. **ServerRoot**: Define o diretório raiz do servidor Apache.

2. **Listen**: Especifica os endereços IP e portas em que o servidor Apache deve escutar.

3. **DocumentRoot**: Define o diretório raiz para os arquivos do site.

4. **Directory**: Configura as opções para um diretório específico, como restrições de acesso, autenticação, etc.

5. **Files**: Configura as opções para um arquivo específico.

6. **VirtualHost**: Define um bloco de configuração para um virtual host específico, permitindo que vários domínios sejam hospedados em um único servidor.

7. **Alias**: Cria um apelido (alias) para um diretório específico.

8. **DirectoryIndex**: Especifica a ordem dos arquivos que serão procurados e exibidos quando um diretório for solicitado.

9. **ErrorLog**: Define o local e o arquivo de log de erros do servidor.

10. **CustomLog**: Define o local e o formato do arquivo de log de acesso do servidor.

11. **Options**: Configura as opções de funcionamento do servidor, como Indexes, FollowSymLinks, etc.

12. **AllowOverride**: Especifica quais diretivas podem ser sobrescritas em arquivos .htaccess.

13. **Require**: Define as regras de autenticação e controle de acesso para um diretório ou localização específica.

Essas são apenas algumas das principais diretivas do Apache. Existem muitas outras diretivas disponíveis que permitem ajustar e personalizar o comportamento do servidor de acordo com suas necessidades específicas. Para obter mais informações sobre as diretivas disponíveis, você pode consultar a documentação oficial do Apache.
